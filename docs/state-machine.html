Device Status Transitions

How the normalizer assigns and evolves device status based on source coverage — JumpCloud, CrowdStrike, Okta

FULLY_MANAGED JC + CS + Okta + owner email
MANAGED JumpCloud + CrowdStrike
NO_EDR JumpCloud enrolled, no CrowdStrike
NO_MDM CrowdStrike present, no JumpCloud
IDP_ONLY Okta only — no MDM or EDR
SERVER CS + server pattern, MDM not required
STALE No activity in >90 days
ACKNOWLEDGED Manually excluded from metrics
DISAPPEARED Removed from all sources (soft-delete)
